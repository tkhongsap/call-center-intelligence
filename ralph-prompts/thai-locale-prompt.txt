Add English (EN) and Thai (TH) locale support to the Call Center Intelligence app.

## Priority Order (complete in sequence):

- Install next-intl package: npm install next-intl
- Create i18n configuration file: src/i18n.ts

- Update next.config.js for i18n support
- Files: package.json, src/i18n.ts, middleware.ts, next.config.js

### 2. Create Translation Files Structure
- Create messages directory: messages/en.json, messages/th.json

## Technical Requirements:
- Use next-intl (recommended for Next.js 14 App Router)
- Locale routing pattern: /en/cases, /th/cases
- Default locale: EN (no prefix required, optional)
- Translation namespace structure: common, navigation, filters, pages
- Use useTranslations() hook in client components
- Use getTranslations() in server components

## Translation Key Naming Convention:
- navigation.home, navigation.alerts, navigation.cases
- common.view, common.share, common.cancel, common.confirm
- filters.allTypes, filters.allSeverities, filters.clearAll
- pages.cases.title, pages.alerts.emptyState
