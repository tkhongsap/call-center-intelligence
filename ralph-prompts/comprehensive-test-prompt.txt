Test all features of the Call Center Intelligence app to ensure there are no errors.

## Testing Approach:
Run the app with `npm run dev` and systematically verify each feature works correctly.
Use curl/API testing and verify responses.

## Priority Order (complete in sequence):

### 1. Build & Startup Verification
- Run `npm run typecheck` - must pass with no errors
- Run `npm run lint` - must pass (ignore pre-existing warnings in .claude/skills/)
- Run `npm run dev` - server must start on port 3000
- Verify http://localhost:3000 redirects properly
- Files to check: package.json, tsconfig.json

### 2. API Endpoint Testing
Test all API endpoints return 200 status:
- GET /api/pulse - returns KPIs data
- GET /api/cases - returns cases list with pagination
- GET /api/cases/:id - returns single case details
- GET /api/alerts - returns alerts list
- GET /api/alerts/:id - returns single alert details
- GET /api/alerts/count - returns alert count
- GET /api/feed - returns feed items
- GET /api/trending - returns trending topics
- GET /api/pulse/sparklines - returns sparkline data
- GET /api/pulse/wordcloud - returns word cloud data
- GET /api/inbox - returns inbox items
- GET /api/uploads - returns upload history
- GET /api/search?q=test - returns search results
- POST /api/chat - responds to chat messages
- Files: app/api/**/*.ts

### 3. Page Route Testing (English)
Verify all English pages load without 500 errors:
- /en/home - Live feed and pulse sidebar
- /en/cases - Cases list with pagination
- /en/cases/:id - Case detail with timeline
- /en/alerts - Alerts list
- /en/alerts/:id - Alert detail with contributing cases
- /en/uploads - Upload interface
- /en/inbox - Inbox list
- /en/trending - Trending topics
- /en/search?q=billing - Search results
- Files: app/[locale]/**/page.tsx

### 4. Page Route Testing (Thai)
Verify all Thai pages load with Thai translations:
- /th/home - Thai navigation labels visible
- /th/cases - Thai UI elements present
- /th/alerts - Thai content renders
- /th/uploads - Thai labels visible
- /th/inbox - Thai translations work
- /th/trending - Thai content renders
- Files: messages/th.json, app/[locale]/**/page.tsx

### 5. Locale-Specific Features
- Test /api/pulse/sparklines?locale=th returns Thai weekday names
- Test /api/pulse/sparklines?locale=en returns English weekday names
- Verify locale parameter works in API
- Files: app/api/pulse/sparklines/route.ts, lib/utils.ts

### 6. Chat Functionality
Test chat API responds correctly:
- POST /api/chat with "show cases" - returns response
- POST /api/chat with "show alerts" - returns response
- Files: app/api/chat/route.ts, lib/chatIntents.ts

### 7. Database Health
- Verify database has seeded data (~2000+ cases, 8+ alerts)
- Test pagination works on cases API
- Files: lib/db/schema.ts, data/call-center.db

## Quality Checks:
- `npm run typecheck` - MUST pass
- `npm run lint` - MUST pass (pre-existing warnings OK)
- `npm run build` - MUST complete successfully
- No 500 errors on any page route

## Completion Criteria:
- All 7 priority items verified
- npm run typecheck passes
- npm run build passes
- All API endpoints return 200
- All page routes load without 500 errors
- Thai locale shows Thai content
- English locale shows English content

Output <promise>ALL FEATURES VERIFIED</promise> when ALL items pass.
